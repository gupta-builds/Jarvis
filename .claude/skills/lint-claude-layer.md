# lint-claude-layer

**Description:** Health-check the `60_Claude/` layer for messiness, duplication, staleness, and weak links.

**Usage:** `/lint-claude-layer`

---

## Instructions

When this skill is invoked:

### 1. Scan the Claude Layer

Read all files in `60_Claude/`:
- Extract frontmatter (`type`, `status`, `created`, `notes`)
- Build a map of links between notes
- Note orphan pages (no inbound links)
- Note hub pages (many inbound links)

### 2. Check for Issues

**Duplication:**
- Search for notes with similar titles/content
- Flag potential duplicates for review

**Staleness:**
- Notes with `status: seed` for >30 days
- Claims that newer sources have superseded
- Notes with no `notes:` backlinks

**Orphans:**
- Notes with no inbound links
- Notes not listed in index

**Gaps:**
- Entities mentioned without pages
- Concepts referenced but not defined
- Questions raised but unanswered

**Weak Links:**
- Notes with only 1 backlink
- Notes not connected to any project

**Index Health:**
- Is `Claude Layer Index.md` up to date?
- Do all notes appear in the index?

### 3. Create Lint Report

Create/update `60_Claude/50_Reviews/Lint Report - YYYY-MM-DD.md`:

```markdown
---
type: review
status: complete
created: YYYY-MM-DD

  - "[[60_Claude Board]]"
---
```
