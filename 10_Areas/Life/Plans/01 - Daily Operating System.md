# Daily Operating System

The one checklist `/today` and `/ops morning-start` paste on top of the day. Every item has a **done definition** and a vault link. If you can only read one note before starting a day, read this one.

## The 5 wins (non-negotiable, with MVP for low-energy days)

| # | Win | Full version | MVP (still counts) | Done when |
|---|-----|--------------|--------------------|-----------|
| 1 | **Physical** | Gym session (5×/week) | 20 push-ups + 20 squats + 5 min stretch | Logged in [[Daily Habit Board]] |
| 2 | **Project** | 1 commit / 1 issue on flagship (Bangalore phase) | Course/hackathon admin step OR skip in Dubai phase | Evidence: commit hash, screenshot, or note path |
| 3 | **Career** | LeetCode ≥5 problems | LeetCode 5 (still required) | Count + topics in Today note → see [[05 - LeetCode & CSCI 4041]] |
| 4 | **Cleanup / admin** | One course assignment OR inbox zero | Smallest admin step (HIST dropbox, 1 WebAssign problem) | Artifact posted/filed |
| 5 | **Review** | Full `/closeday` | 5-min end-of-day note | Today note's review block filled |

> Career win = LeetCode by default. Internship-pipeline work is a **weekly** cadence (see [[02 - Weekly Operating System]]), not a daily minimum.

## Default day shape (Dubai week — adapt to calendar)

Today's calendar drives the *when*; this is the fallback when it's empty. Capacity target: **3–5 h focused**.

| Block | Time (Dubai) | Use |
|-------|--------------|-----|
| Morning launch | 30 min | Open Today note; no phone/doomscroll before first task ([[08 - Anti-Drift Rules]]) |
| Deep work 1 | 60–90 min | Hardest academic track of the day (usually MATH 2230 or 2033) |
| LeetCode | 45–60 min | 5 problems, one topic ([[05 - LeetCode & CSCI 4041]]) |
| Deep work 2 | 45–60 min | CSCI 4041 review OR project (Bangalore phase) |
| Admin | 20–30 min | Course dropbox / WebAssign / inbox |
| Physical | 45–60 min | Gym or calisthenics |
| Evening close | 15 min | `/closeday` → review win |

## Done definition for the *day*

A day is **green** when ≥90% of the academic-stack rows + all 5 wins (full or MVP) are checked. `/closeday` computes this; if red, it writes **one** friction fix for tomorrow (see [[08 - Anti-Drift Rules]]).
