# Weekly Operating System

The Mon–Sun rhythm that carries from Dubai into Bangalore. Daily ops ([[01 - Daily Operating System]]) is the engine; this is the gearbox — it sets the week's one flagship priority and verifies the week produced evidence.

## Weekly rhythm (Bangalore execution shape)

| Day | Focus |
|-----|-------|
| **Mon** | Pick **one** flagship priority (shippable in 7 days). Write acceptance criteria. Set LeetCode topic week. |
| **Tue–Thu** | Build / study. Daily commit + bug log. If blocked, cut scope **same day** — don't carry dead work. |
| **Fri** | Produce **evidence**: README update + screenshot/demo + 1 portfolio/interview bullet. |
| **Sat** | Stress-test: "where does it break?" pass + add one failure test/trace point. (Hackathon block if calendar forces it.) |
| **Sun** | **Weekly review** (7 questions below) + course deadline scan + pick next week's flagship. |

In **Dubai phase** the "flagship priority" is usually *installing the routine itself* or *the CSCI 2033 LA base* — not a product. See [[03 - Monthly & Phase Map]].

## Sunday weekly review — 7 questions (verbatim from Summer Grind)

> Answer inside the weekly note (`60_Claude/50_Reviews/Weekly Review YYYY-WXX.md`):
>
> 1. What did I ship?
> 2. What did I train physically?
> 3. What did I do for money/career?
> 4. What did I clean or automate?
> 5. What did I read or learn?
> 6. Which habit broke, and how do I make the next rep easier?
> 7. What is the one flagship priority for next week?

## Weekly course deadline scan (every Sunday)

Re-read the deadline table in [[04 - Summer Courses Ops]]. Flag everything due in the **next 7 days** and pre-load it into Mon–Sun:

- Any **MATH 2230** test → block 2 prep sessions before the date.
- Any **HIST 1103** assignment/exam → block one writing session the day before (admin-only).
- Any **WebAssign** HW/quiz due → assign to a weekday admin slot.

## Weekly evidence (the objective on-track signal)

A week is **on track** if it produced:

- A short **"What shipped?"** line (even tiny) — Friday evidence.
- **LeetCode ≥35** for the week (5/day floor) — summed from Today notes.
- **≥1 MATH 2230 ML concept note** ([[06 - ML Fundamentals (2033 + 2230)]]).
- All course items due that week submitted on time.
- **Internship pipeline** touched once (the weekly career cadence): 1–3 applications or one follow-up in the tracker. This is the weekly career task that daily LeetCode does **not** cover.

## How this wires into ops

`/closeday` rolls daily scorecards up; the Sunday run does the 7 questions + deadline scan and writes next week's flagship into Monday's Today note.
