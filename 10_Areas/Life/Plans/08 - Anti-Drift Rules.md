# Anti-Drift Rules

The named failure mode: **deviating whenever a tab switches**, and **MCP/tool tinkering instead of execution**. Infra is built — setting up AI is an infinite process and you've realized it. These rules channel effort instead of moralizing tool use.

## The core rules

1. **No platform archaeology this session or in daily ops.** No choosing MCPs, editing `.mcp.json`, comparing AI tools, "optimizing the stack." Use existing Jarvis MCP + calendar; let the AI pick its own tooling. Tool work is allowed only as an explicit, time-boxed weekly slot — never mid-task.
2. **First task before dopamine.** No phone / doomscroll / smoking before planning + breakfast + first task (from [[Summer Grind]] routine).
3. **One tab, one task.** When you catch a tab-switch toward setup/optimization, that's the cue — return to the open Today note. The Today note is the single source of "what now."
4. **Never Miss Twice** ([[Atomic Habits]]). Missing once is data. Missing twice starts the wrong system. After any miss, the next rep gets *easier*, not skipped.
5. **Tool tinkering is time-boxed.** If you genuinely need a tool change, write it to the parking lot (below) and do it in the weekly tool slot — not now.

## The "Do NOT do today" list (goes in every Today note)

- MCP / connector setup or comparison
- New agents / new skills authoring
- GitHub repo stars triage
- AI platform comparison / stack optimization
- ".obsidian" / plugin configuration

If the urge hits, it goes to the parking lot, not the day.
