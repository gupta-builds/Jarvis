- **Low external dependency risk:** unlike trading, you’re not bottlenecked by market performance or “alpha”; success is product + engineering quality.
- **Fits your daily rule:** it naturally generates “one project win” every day (issue-based progress).

“I help students convert study time into weekly progress insight + a personalized tutor by tracking sessions and summarizing them.”

### Hero flow (keep it brutally simple)
- Log in → start session timer → end session + quick notes → weekly recap dashboard → ask “tutor” questions grounded in your sessions.

### Success metric (pick 1)

## What *not* to make flagship (80/20 reasons)
### Trading App as flagship: high effort, lower predictable payoff
It’s valuable, but the “product success” is fuzzy and evaluation is complicated. Great as a **supporting** project for ML/MLOps credibility once Arc is shipping.

### Jarvis as flagship: can balloon into endless tooling
Jarvis is powerful, but it’s very easy for “ops layer” work to eat the summer while producing less legible portfolio proof. Keep it in **maintenance lane** with strict boundaries.

## The 80/20 portfolio stack (one flagship + one supporting + one maintenance)
- **Flagship (80%): Arc (Learning Tracker)** — 6–10 weeks to a polished v1.
- **Supporting (15–20%): Trading App (evaluation-focused)** — build only the honest pipeline + backtest + dashboard.
- **Maintenance (5%): Jarvis + Laptop/Phone automation** — only items that directly reduce friction for Arc + applications.
