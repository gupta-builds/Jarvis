# Unsloth

**What it is:** A fine-tuning acceleration library and web UI (Unsloth Studio) for training open models like Gemma 4, Qwen3, DeepSeek locally — 2-5x faster than standard training with 70% less memory.

**Why it's here:** Useful for UROP/research work involving fine-tuning open models on domain-specific data.

**Why it's not a priority:** Unsloth is for training models, not for using Claude. There is no Claude integration — the workflow is entirely orthogonal to the Claude Code + Jarvis stack. Revisit if BOOM/UROP work requires fine-tuning a custom model on proprietary data that can't go to Anthropic's API.
